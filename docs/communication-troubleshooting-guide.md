# 통신 장애진단 가이드 (2단계)

## RS-485

데이지체인 권장, 스타배선 주의, A/B 표기 혼동, 종단·바이어스·차폐·공통전위.

## Modbus RTU

Slave ID, FC, 0/1-based 주소, CRC, timeout, byte/word order. 제조사 레지스터는 매뉴얼 없이 작성 금지.

## 점검 순서

전원 → 케이블 → 극성 → 접지/차폐 → 토폴로지 → 종단 → 설정 → ID → 주소 → 요청/응답 → CRC → 타임아웃 → 폴링 → 노이즈 → 절연 → 로그 → 대체시험

인터넷 장애 시에도 필수 제어는 현장 PLC에서 유지. PLC 인터넷 직접 노출 비권장.
